# インタビュー質問集

各ノートを対話的に埋めるための質問例。すべてを聞く必要はなく、サービスの段階や複雑さに応じて取捨選択する。

## 質問のスタイル

- **オープン質問から始める**: 「どんなサービスにしたいですか?」
- **クローズド質問で詰める**: 「Free と Pro の二段階、それとも Free / Pro / Enterprise の三段階?」
- **選択肢を提示する**: ユーザーが決めかねている時は具体的な選択肢を出す
- **数字を引き出す**: 「月 100 人くらい?」「100 円 / 500 円 / 1000 円のどれくらい?」

## 一度に聞く量

質問は **1 回に 1〜3 個** まで。ask_user_input_v0 ツールを使えるなら積極的に使う（選択肢を提示できるので考えやすい）。

質問を浴びせ続けない。1 つの質問の答えが出てから次に進む。

# 既存サービス整理モード用の質問

新規ではなく既存サービスを整理するときの初期質問:

- 「既存サービスの情報源は何が一番詳しい?（README / Obsidian / Notion / 口頭 / Slack のスレッド)」
- 「ドキュメント化されてない、頭の中にしかない情報も多い?」
- 「いま整理したい一番の動機は?（新メンバーへの引き継ぎ / 自分の整理 / 投資家向け / etc）」
- 「優先度の高いノートから始めるなら、どれから?」

そこからは共通骨格を一つずつ「これって現状どうなってる?」と聞いていく形。

# サービス固有論点の察知と提案

固有論点に気づいたら、以下のような形で切り出す:

- 「いま話してる○○、これかなり重要そうですね。独立したノートとして残しません?」
- 「○○ の決定、サービスのコアにかかわる判断ですよね。ノート化しときましょうか?」
- 「これって他のサービスに当てはまらない、このサービス固有の論点だと思います。残しときます?」

ユーザーが NO と言ったら作らない。YES と言ったらノートを作る。

「もう少し決まってから」と言われたら一旦保留にして、また話題が戻ってきたら再提案。
